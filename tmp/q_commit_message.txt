Fix(events): Improve error message display on event edit form

- Modified the `handleSubmit` function in
  `frontend/src/app/dashboard/events/[eventId]/edit/page.tsx`
  to better parse backend error responses.
- If `errorData.detail` from the backend is an array (typical for
  FastAPI validation errors), the message from the first error
  object is extracted and displayed.
- If `errorData.detail` is a string, it's used directly.
- If `errorData.detail` is another non-string type, it's stringified.
- This prevents the display of `[object Object]` when validation
  errors occur, showing a more user-friendly message instead.
- Updated `.Q/projectlog.md`.
